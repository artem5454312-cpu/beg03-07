# FitPulse — AI-тренер (MVP)

Рабочий каркас приложения по твоему ТЗ: сервер + база данных + AI-агент на Claude +
чаты в реальном времени + PWA-фронтенд в стиле FitPulse (тот же дизайн, что в лендинге:
уголь/холст/кобальт, Bebas Neue + Inter + Space Mono).

Это не игрушечный прототип — все основные потоки реально работают и пишут в базу.
Но некоторые части ТЗ сделаны в упрощённом виде на старте (список в конце файла),
чтобы можно было быстро запустить и показать друзьям-тестерам.

Агент — это обычный вызов Claude API прямо из твоего Node-сервера (никаких отдельных
облачных песочниц/агентов не нужно), но с "инструментами" (tool use): Claude сам решает,
когда вызвать функцию `create_workout`, `add_workout_result` или `update_memory` — и эти
функции сразу пишут в твою таблицу Postgres. Поэтому план тренировок заполняется сам,
без ручного переноса. Весь промпт и инструменты — в одном файле:
`server/services/agentService.js`.

## 0. Быстрый старт на Railway (если нужен только этот шаг)

1. Открой свой проект в Railway.
2. Кликни на **сервис приложения** (не на Postgres).
3. Вкладка **Variables** → **New Variable**.
4. Name: `ANTHROPIC_API_KEY`, Value: твой ключ с console.anthropic.com/settings/keys.
5. Ещё одна переменная: `PROMO_CODE` со значением `5680` — это PIN на входе, пока
   приложение в разработке (см. раздел про экран PIN ниже).
6. Нажми Add — Railway сам передеплоит сервис с новыми переменными.

**Важно:** ничего создавать в разделе "Managed Agents" Claude Console не нужно — это
отдельный продукт Anthropic для автономных агентов в облачных песочницах (пишут код,
лазят по интернету часами, тарификация по часам сессии). Твоему агенту это не подходит
и не нужно: он живёт прямо внутри твоего же Node-сервера как обычный вызов API.

## 1. Что внутри

```
ai-trainer/
  server/            ← бэкенд (Node.js + Express)
    index.js          — точка входа, поднимает сервер
    schema.sql         — вся структура базы данных
    initDb.js          — команда, которая создаёт таблицы
    ws.js               — чаты в реальном времени (WebSocket)
    scheduler.js         — фоновые напоминания
    routes/               — все ручки API (auth, agent, plan, chats, profile, admin)
    services/agentService.js — общение с Claude + "инструменты" агента
    services/push.js          — push-уведомления в браузер
  public/            ← фронтенд (то, что видит пользователь), PWA
    index.html, css/, js/     — само приложение (агент/план/чаты/профиль)
    admin-panel.html           — скрытая админка (её нет в меню приложения)
    manifest.json, sw.js        — чтобы можно было "добавить на экран домой"
  .env.example        ← шаблон настроек, скопируешь в .env
  package.json
```

## 2. Куда вставлять ключи и данные

Всё, что нужно вписать — в одном месте: файл `.env` (создаётся из `.env.example`).

1. Скопируй файл: `.env.example` → переименуй копию в `.env`
2. Открой `.env` и заполни по порядку:

| Поле | Что это | Где взять |
|---|---|---|
| `DATABASE_URL` | адрес базы данных PostgreSQL | Railway → твой Postgres-сервис → вкладка **Connect** → "Postgres Connection URL" |
| `ANTHROPIC_API_KEY` | ключ для AI-агента | https://console.anthropic.com/settings/keys |
| `JWT_SECRET`, `JWT_REFRESH_SECRET` | секретные строки для входа в аккаунт | просто придумай два разных длинных случайных набора символов (команда для генерации есть прямо в `.env.example`) |
| `PROMO_CODE` | промокод для закрытого теста | придумай любое слово, дай его друзьям |
| `ADMIN_PASSWORD` | пароль в скрытую админку | придумай, никому не давай |
| `VAPID_PUBLIC_KEY` / `VAPID_PRIVATE_KEY` | для push-уведомлений в браузере | команда `npx web-push generate-vapid-keys` (см. ниже) — необязательно на старте, можно оставить пустым |

Больше **нигде в коде** ключи вписывать не нужно — всё остальное само подхватывает их из `.env`.

## 3. Как запустить у себя на компьютере (перед Railway)

Нужен установленный Node.js (версия 18 и выше) и доступ к любой PostgreSQL-базе
(можно временно взять бесплатную на Railway, Neon или Supabase — просто скопировать её адрес в `DATABASE_URL`).

```bash
cd ai-trainer
npm install                # ставит все библиотеки
cp .env.example .env       # создать файл настроек
# теперь открой .env и впиши свои ключи (см. таблицу выше)

npm start                  # запускает сервер — таблицы в базе создаются сами при старте
```

Открой в браузере `http://localhost:3000` — увидишь экран с промокодом.
Админка — на `http://localhost:3000/admin-panel.html` (её специально нет в обычном меню).

## 4. Как выложить на Railway

1. Залей папку `ai-trainer` в GitHub-репозиторий.
2. В Railway: **New Project → Deploy from GitHub repo**, выбери репозиторий.
3. Добавь в этот же проект **PostgreSQL** (кнопка "+ New" → Database → PostgreSQL).
4. В настройках своего сервиса (не базы) открой вкладку **Variables** и вставь туда
   все значения из своего `.env` — включая `DATABASE_URL`, который Railway обычно
   может подставить сам через ссылку на базу (в Variables можно нажать "Add Reference").
5. Railway сам определит команду `npm start` из `package.json` — ничего дополнительно
   настраивать не нужно. Таблицы в базе создаются автоматически при первом запуске
   сервера, вручную ничего выполнять не нужно.

Готово — по адресу, который выдаст Railway, будет открываться приложение.

## 5. Как это устроено (по-простому)

- **Вход** — вместо промокода теперь экран "Введите PIN" (приложение помечено как
  "в разработке"). Технически это та же переменная `PROMO_CODE` (поставь `5680` или любое
  другое значение в Railway Variables) — просто по-другому называется в интерфейсе.
- **Экран "Агент"** — это чат с Claude. При каждом сообщении сервер подмешивает в разговор
  два вида "памяти": (1) последние сообщения переписки и (2) отдельно сохранённые факты
  о тебе (цель, уровень, ограничения) — они не стираются кнопкой "Очистить диалог".
  Агент знает сегодняшнюю дату и день недели, поэтому если ты называешь срок ("у меня
  16 дней", "распиши на 2 недели") — он обязан расписать тренировки на весь этот срок
  (а не 2-3 штуки для примера), и вызывает команды: добавить тренировку в план, обновить
  факты о тебе, записать результат, сохранить краткое описание плана — это "инструменты"
  (tool use), и работают они прямо внутри твоего Node-сервера, без каких-либо дополнительных
  облачных сервисов.

- **Экран "План"** — тренировки собраны блоками по целям (бег отдельно, зал отдельно —
  не мешаются в одном списке). У каждого блока: заголовок от агента, счётчик выполненных,
  диапазон дат и кнопка удаления всего блока. Клик по тренировке открывает карточку с
  деталями и кнопкой **"Начать тренировку"** — она открывает таймер (обратный отсчёт с
  паузой/продолжением, +1/−1 минута), а "Завершить" сразу отмечает тренировку выполненной.
  Статусы теперь три: В процессе / Выполнено / Пропущено. Тренировки, которые прошли
  больше 2 дней назад и не отмечены, раз в сутки автоматически помечаются "пропущено".

- **Экран "Чаты"** — один общий чат на город (создаётся автоматически, когда кто-то
  указывает этот город в профиле) через WebSocket, то есть сообщения появляются мгновенно
  без обновления страницы. Сообщения показаны как пузырьки с аватаром (или буквой имени,
  если аватара нет), свои сообщения — синие и справа, чужие — серые и слева.

- **Экран "Профиль"** — можно загрузить аватар с телефона (кружок сверху): картинка
  сжимается прямо в браузере до маленького квадрата и сохраняется как часть профиля —
  отдельное файловое хранилище (S3) для этого не требуется. Аналитика показывает и общую
  статистику за всё время, и отдельно по текущей цели.

- **Админка** (`/admin-panel.html`) — не в общем меню, доступна только по прямой ссылке
  и отдельному паролю (`ADMIN_PASSWORD`), пароль проверяется на сервере, а не в браузере.

## 6. Что сделано в упрощённом виде на этом этапе

Чтобы не тормозить первый запуск, эти пункты из ТЗ пока заглушены или упрощены —
дальше их можно докрутить по одному:

- **Вход по телефону/SMS** — сейчас вход по логину/паролю + промокод. SMS требует
  подключения стороннего сервиса (например, Twilio) и отдельного шага, чтобы не тратить
  время на старте.
- **Push-уведомления** — код готов (`services/push.js`), но реально начнёт слать
  уведомления только после того, как впишешь `VAPID_PUBLIC_KEY` / `VAPID_PRIVATE_KEY`.
  Также стоит учесть: на iPhone (Safari) у push для PWA есть свои ограничения — это
  отдельно нужно будет проверить на реальном устройстве.
  До этого момента напоминания просто сохраняются в базу (раздел "Уведомления" в API есть).
- **Проактивные сообщения агента "как прошла тренировка"** — сама схема (фоновый
  планировщик) в `scheduler.js` заложена, но сейчас она шлёт напоминание по шаблону,
  а не зовёт Claude каждый раз — это специально сделано, чтобы не тратить деньги на
  API без ограничения частоты (в ТЗ это тоже отмечено как открытый вопрос).
- **Модерация чата** — есть только кнопка "пожаловаться" (жалобы видно в админке),
  автоматического бана/фильтра спама пока нет.
- **Файлы** (фото профиля, файлы результатов тренировок) — в базе для них уже есть
  поля (`photo_url`, `file_url`), но самой загрузки файлов в облако (S3) пока нет —
  сейчас это просто текстовые ссылки. Подключить реальную загрузку — следующий шаг.
- **Согласие на обработку данных** — на экране регистрации пока нет чекбокса-оферты,
  стоит добавить перед тем, как звать тестеров.
- **Один город на пользователя** — как и было в ТЗ на старте: пока нельзя состоять
  сразу в нескольких городских чатах.

Все эти пункты не требуют переделки архитектуры — они добавляются поверх того, что уже есть.

## 7. Если что-то не запускается

- Ошибка про `ANTHROPIC_API_KEY` при отправке сообщения агенту → проверь, что ключ
  вписан в `.env` и что после правки `.env` сервер был перезапущен.
- Ошибка подключения к базе → проверь `DATABASE_URL`, и что `npm run db:init` был
  выполнен хотя бы один раз.
- Чат не приходит мгновенно → проверь в консоли браузера (F12), что WebSocket
  подключился без ошибок 401 (значит просрочен токен входа — перелогинься).
